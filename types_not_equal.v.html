When are Coq types provably unequal?

Coq's dependent type system is very expressive, but also often annoying to use. One reason why is that the type checker isn't good at proving types unequal. For example, Coq rejects the following definition of f:

Inductive T : Set → Type :=
| C1 : T ℕ
| C2 : T 𝔹.

Fail Definition f (x : T ℕ) : unit :=
  match x with
  | C1 ⇒ tt
  end.

⟹ 
  The command has indeed failed with message: 
  Non exhaustive pattern-matching: no clause found for pattern C2
Coq doesn't realize that the C2 case is impossible. Meanwhile, OCaml accepts the analogous program (written using GADTs) without complaining about non-exhaustive pattern matching: 
  type 'a t
    = C1 : int t
    | C2 : bool t
  let f (x : int t) : unit =
    match x with
    | C1 -> ()
To do so, it has to reason that the C2 case is impossible because int is not equal to 𝔹. (Since exhaustiveness checking is undecidable for GADTs, it's possible that the OCaml compiler just gave up on trying to analyze the match expression. But that's unlikely, because it does complain about exhaustivity if I add an extra constructor C3 : int t.)
We could convince Coq that C2 is impossible if we had a proof that 𝔹 is not equal to ℕ:

Definition f (H : 𝔹 ≠ ℕ) (x : T ℕ) : unit :=
  match x in T A return A = ℕ → unit with
  | C1 ⇒ λ _ ⇒ tt
  | C2 ⇒ λ Heq : 𝔹 = ℕ ⇒ match H Heq with end
  end eq_refl.

But how to prove two types unequal? The usual tactics for proving things unequal (discriminate, inversion, etc.) don't work because ℕ and 𝔹 aren't constructors.
Some metatheory handwaving suggests that proving two types unequal will be impossible for most practical situations. If we could prove A ≠ B for isomorphic A and B, then univalence wouldn't be independent of CiC. So we should expect to be able to prove A ≠ B only when A and B aren't isomorphic. This is pretty bad: most data types in functional programming are trees of some kind or other, which are all isomorphic to each other (they all have a countable number of inhabitants).
But, let's try to prove whatever we can anyway. We can define a type A as less than or equal to B if there is an injection f : A → B:

Definition injective {A B} (f : A → B) := ∀ x y, f x = f y → x = y.

Definition leq A B := ∃ f : A → B, injective f.
Infix "⊑" := leq (at level 70, no associativity).

Lemma leq_refl {A} : A ⊑ A.

Then, two types are isomorphic if they're less than or equal to each other:

Definition iso A B := A ⊑ B ∧ B ⊑ A.
Infix "≅" := iso (at level 70, no associativity).
Notation "a '≇' b" := (¬ (a ≅ b)) (at level 70, no associativity).

Lemma iso_refl {A} : A ≅ A.

To show two types unequal, it's sufficient to show that they aren't isomorphic:

Polymorphic Lemma iso_ne {A B} : A ≇ B → A ≠ B.

This is already enough to prove ℕ ≠ 𝔹: ℕ and 𝔹 aren't isomorphic because 𝔹 has two inhabitants while ℕ has countably many.

Lemma nat_ne_bool : ℕ ≠ 𝔹.
Proof.
  apply iso_ne; intros [[f Hfg] [g Hgf]].
  f : nat -> bool
  Hfg : injective f
  g : bool -> nat
  Hgf : injective g
  ============================
  False
Since f returns a boolean, [f 0, f 1, f 2] must contain a duplicate.
  pose proof Hfg 0 1 as H0.
  pose proof Hfg 1 2 as H1.
  pose proof Hfg 0 2 as H2.
  destruct (f 0), (f 1), (f 2); firstorder congruence.
Qed.

It'd be nice to automate this kind of reasoning for more complicated types using reflection. To do so, we'll first need a bunch of helper definitions and lemmas about (⊑) and (≅).
A type is inhabited if we can construct a value of that type:

Definition inhabited A := ∃ x : A, ⊤.

A type is finite if it has exactly n inhabitants for some n : ℕ. We can represent a finite type with n inhabitants as a sum ⊤ + (⊤ + (.. + ⊥)):

Definition fin n : Type := Nat.iter n (sum ⊤) ⊥.

For example, 𝔹 is a finite type with 2 inhabitants:

Lemma fin_bool : 𝔹 ≅ fin 2.

It's easy to decide whether two finite types are equal, because finite types are isomorphic iff they have the same number of inhabitants:

Lemma fin_leq {m n} : fin n ⊑ fin m ↔ n ≤ m.
Lemma fin_iso {m n} : fin n ≅ fin m ↔ n = m.

So now we know how to prove/disprove A ≠ B in the case where A and B are both finite. But what to do if one of the types is infinite?
First, we can define A as strictly smaller than B if there's no injection f : B → A:

Definition lt A B := ¬ B ⊑ A.
Infix "⊏" := lt (at level 70, no associativity).

Every fin n is strictly smaller than ℕ, and A is strictly smaller than A → fin 2 by diagonalization:

Lemma fin_lt_nat {n} : fin n ⊏ ℕ.
Lemma A_lt_PA {A} : A ⊏ (A → fin 2).

Interestingly, if A is infinite, then changing the codomain from fin 2 to fin (2 + n) doesn't make the cardinality any bigger:

Lemma PA2n_eq_PA {A n} : inhabited A → (∀ k, fin (S k) × A ≅ A) →
  (A → fin (2 + n)) ≅ (A → fin 2).

In other words, the following types are all isomorphic:
  • ℕ → fin 2
  • ℕ → fin 3
  • ℕ → fin 4
  • ℕ → fin 5
  • ...
In fact, the above types are even isomorphic to ℕ → ℕ. In general, going from A → fin (2 + n) to A → ℕ doesn't change the cardinality as long as A is big enough:

Lemma PAnat_eq_PA {A} : inhabited A → ℕ × A ⊑ A →
  (A → ℕ) ≅ (A → fin 2).

Inductive nf :=
| Finite (n : ℕ)
| Tower (n : ℕ).

Fixpoint tower n :=
  match n with
  | 0 ⇒ Nat
  | S n ⇒ Fun (tower n) (Fin 2)
  end.

Fixpoint nfD t :=
  match t with
  | Finite n ⇒ Fin n
  | Tower n ⇒ tower n
  end.

Lemma tower_succ n : ⊤ + ⟦tower n⟧ ≅ ⟦tower n⟧.
Lemma tower_add_fin m n : fin m + ⟦tower n⟧ ≅ ⟦tower n⟧.

Fixpoint norm t : nf :=
  match t with
  | Fin n ⇒ Finite n
  | Nat ⇒ Tower 0
  | Add t1 t2 ⇒
    match norm t1, norm t2 with
    | Finite n, Finite m ⇒ Finite (n + m)
    | Tower n, Tower m ⇒ Tower (max m n)
    | _, Tower n | Tower n, _ ⇒ Tower n
    end
  | Mul t1 t2 ⇒
    match norm t1, norm t2 with
    | Finite 0, _ | _, Finite 0 ⇒ Finite 0
    | Finite n, Finite m ⇒ Finite (n × m)
    | Tower n, Tower m ⇒ Tower (max m n)
    | _, Tower n | Tower n, _ ⇒ Tower n
    end
  | Fun t1 t2 ⇒
    match norm t1, norm t2 with
    | Finite 0, _ | _, Finite 1 ⇒ Finite 1
    | _, Finite 0 ⇒ Finite 0
    | Finite m, Finite n ⇒ Finite (Nat.pow n m)
    | Finite _, Tower n ⇒ Tower n
    | Tower n, Finite (S (S _)) ⇒ Tower (S n)
    | Tower n, Tower 0 ⇒ Tower (S n)
    | Tower n, Tower (S m) ⇒ Tower (S (max n m))
    end
  end.

Lemma norm_spec t : ⟦t⟧ ≅ ⟦nfD (norm t)⟧.

Definition nf_iso t1 t2 :=
  match t1, t2 with
  | Finite n, Finite m ⇒ Nat.eqb n m
  | Tower n, Tower m ⇒ Nat.eqb n m
  | _, _ ⇒ false
  end.

Lemma nf_iso_spec t1 t2 : Bool.reflect (⟦nfD t1⟧ ≅ ⟦nfD t2⟧) (nf_iso t1 t2).

Definition isob t1 t2 := nf_iso (norm t1) (norm t2).

Lemma isob_spec t1 t2 : Bool.reflect (⟦t1⟧ ≅ ⟦t2⟧) (isob t1 t2).

Class Reifies A t := reifies : A ≅ ⟦t⟧.
Instance reify_nat : Reifies ℕ Nat := iso_refl.
Instance reify_bool : Reifies 𝔹 (Fin 2) := fin_bool.
Instance reify_sum {A B s t} `{HA : Reifies A s} `{HB : Reifies B t} :
  Reifies (A + B) (Add s t).

Instance reify_fun {A B s t} `{HA : Reifies A s} `{HB : Reifies B t} :
  Reifies (A → B) (Fun s t).
