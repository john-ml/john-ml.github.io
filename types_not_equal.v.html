When are Coq types provably unequal?

When I first started learning Coq, I expected it to be strictly more expressive than languages with "dependent types at home" (e.g. GADTs, type families, ..). But it's actually pretty difficult to translate code using things like GADTs into Coq, and it seems to be because other languages bake a number of extra rules about type inequality into the type checker.
For example, OCaml accepts the following code without complaining about non-exhaustive pattern matching: 
  type 'a t
    = C1 : int t
    | C2 : bool t
  let f (x : int t) : unit =
    match x with
    | C1 -> ()
To do so, it has to reason that the C2 case is impossible because int is not equal to bool. (Since exhaustiveness checking is undecidable for GADTs, it's possible that the OCaml compiler just gave up on trying to analyze the match expression. But that's unlikely, because it does complain about exhaustivity if I add an extra constructor C3 : int t.)
The seemingly obvious fact that nat is not the same as bool is not obvious to Coq:

Inductive T : Set → Type :=
| C1 : T nat
| C2 : T bool.

Fail Definition failure (x : T nat) : unit :=
  match x with
  | C1 ⇒ tt
  end.

⟹ 
  The command has indeed failed with message: 
  Non exhaustive pattern-matching: no clause found for pattern C2
If we could prove bool ≠ nat, some dependent pattern matching can prove that C2 is impossible:

Definition success (H : bool ≠ nat) (x : T nat) : unit :=
  match x in T A return A = nat → unit with
  | C1 ⇒ λ _ ⇒ tt
  | C2 ⇒ λ Heq : bool = nat ⇒ match H Heq with end
  end eq_refl.

But how to prove two types unequal? The usual tactics for proving things unequal (discriminate, inversion, etc.) don't work because nat and bool aren't constructors.
Some metatheory handwaving suggests that proving two types unequal will be impossible for most practical situations. If we could prove A ≠ B for isomorphic A and B, then univalence wouldn't be independent of CiC. So we should expect to be able to prove A ≠ B only when A and B aren't isomorphic. This is pretty bad: most data types in functional programming are trees of some kind or other, which are all isomorphic to each other (they all have a countable number of inhabitants).
But, let's try to prove whatever we can anyway. We'll say a type A is less than or equal to B if there exists an injection f : A → B:

Definition injective {A B} (f : A → B) := ∀ x y, f x = f y → x = y.

Definition leq A B := ∃ f : A → B, injective f.
Infix "⊑" := leq (at level 70, no associativity).

Lemma leq_refl {A} : A ⊑ A.

Two types are isomorphic if they're less than or equal to each other:

Definition iso A B := A ⊑ B ∧ B ⊑ A.
Infix "≅" := iso (at level 70, no associativity).
Notation "a '≇' b" := (¬ (a ≅ b)) (at level 70, no associativity).

Lemma iso_refl {A} : A ≅ A.

Polymorphic Lemma iso_ne {A B} : A ≇ B → A ≠ B.

This is already enough to prove nat ≠ bool: nat and bool can't be isomorphic because bool has two inhabitants while nat has countably many.

Lemma nat_ne_bool : nat ≠ bool.
Proof.
  apply iso_ne; intros [[f Hfg] [g Hgf]].
  f : nat -> bool
  Hfg : injective f
  g : bool -> nat
  Hgf : injective g
  ============================
  False
Since f returns bool, [f 0, f 1, f 2] must contain a duplicate.
  pose proof Hfg 0 1 as H0.
  pose proof Hfg 1 2 as H1.
  pose proof Hfg 0 2 as H2.
  destruct (f 0), (f 1), (f 2); firstorder congruence.
Qed.

It'd be nice to automate this kind of reasoning for more complicated types. First, some lemmas to make (⊑) and (≅) easier to work with:

Axiom FChoice : Coq.Logic.ChoiceFacts.FunctionalChoice.
Axiom LEM : Coq.Logic.ClassicalFacts.excluded_middle.

Definition surjective {A B} (f : A → B) := ∀ y, ∃ x, f x = y.

Definition inhabited A := ∃ x : A, ⊤.

Definition comp {A B C} (f : B → C) (g : A → B) := λ x ⇒ f (g x).

Lemma leq_asym {A B} : A ⊑ B → B ⊑ A → A ≅ B.
Lemma leq_trans {A B C} : A ⊑ B → B ⊑ C → A ⊑ C.
Lemma leq_False {A} : ⊥ ⊑ A.
Lemma leq_sum1 {A B C} : A ⊑ B → A + C ⊑ B + C.
Lemma leq_sum2 {A B C} : B ⊑ C → A + B ⊑ A + C.
Lemma leq_prod1 {A B C} : A ⊑ B → A × C ⊑ B × C.
Lemma leq_prod2 {A B C} : B ⊑ C → A × B ⊑ A × C.
Lemma leq_fun1 {A B C} : inhabited A → A ⊑ B → (A → C) ⊑ (B → C).

Lemma sum_fun_leq_fun_sum {A B C} :
  inhabited B →
  A + (B → C) ⊑ (B → A + C).

Add Parametric Relation : Type iso
  reflexivity proved by @iso_refl
  symmetry proved by @iso_sym
  transitivity proved by @iso_trans
  as iso_rel.

  signature iso ==> iso ==> iso as iso_sum.

  signature iso ==> iso ==> iso as iso_fun.

A type is finite if it has exactly n inhabitants for some n : nat.

Definition fin n : Type := Nat.iter n (sum ⊤) ⊥.

Lemma fin_False : ⊥ ≅ fin 0.
Lemma fin_True : ⊤ ≅ fin 1.
Lemma fin_bool : bool ≅ fin 2.
Lemma fin_sum {m n} : fin m + fin n ≅ fin (m + n).
Lemma fin_prod {m n} : fin m × fin n ≅ fin (m × n).
Lemma fin_fun {m n} : (fin m → fin n) ≅ fin (Nat.pow n m).

Lemma nat_fin_sum {n} : fin n + nat ≅ nat.
Lemma nat_fin_prod {n} : fin (S n) × nat ≅ nat.
Lemma nat_fin_fun {n} : (fin (S n) → nat) ≅ nat.

Two finite types are equal iff they have the same number of inhabitants:

Lemma fin_leq {m n} : fin n ⊑ fin m ↔ n ≤ m.
Lemma fin_iso {m n} : fin n ≅ fin m ↔ n = m.

A is strictly smaller than B if there's no injection f : B → A:

Definition lt A B := ¬ B ⊑ A.
Infix "⊏" := lt (at level 70, no associativity).

Every fin n is strictly smaller than nat, and A is strictly smaller than A → fin 2 by diagonalization:

Lemma fin_lt_nat {n} : fin n ⊏ nat.

Lemma A_lt_PA {A} : A ⊏ (A → fin 2).

Interestingly, if A is infinite, then changing the codomain from fin 2 to fin (2 + n) doesn't make the cardinality any bigger:

Lemma PA2n_eq_PA {A n} :
  inhabited A →
  (∀ k, fin (S k) × A ≅ A) →
  (A → fin (2 + n)) ≅ (A → fin 2).

In fact, even going from fin (2 + n) to nat doesn't change the cardinality if the domain is big enough:

Lemma PAnat_eq_PA {A} :
  inhabited A →
  A × nat ⊑ A →
  (A → nat) ≅ (A → fin 2).

Inductive type :=
| Fin (n : nat)
| Nat
| Add (t1 t2 : type)
| Mul (t1 t2 : type)
| Fun (t1 t2 : type).

Reserved Notation "'⟦' t '⟧'".
Fixpoint typeD t :=
  match t with
  | Fin n ⇒ fin n
  | Nat ⇒ nat
  | Add t1 t2 ⇒ ⟦t1⟧ + ⟦t2⟧
  | Mul t1 t2 ⇒ ⟦t1⟧ × ⟦t2⟧
  | Fun t1 t2 ⇒ ⟦t1⟧ → ⟦t2⟧
  end%type
where "'⟦' t '⟧'" := (typeD t).


Fixpoint card t : option nat :=
  match t with
  | Fin n ⇒ Some n
  | Nat ⇒ None
  | Add t1 t2 ⇒ match card t1, card t2 with Some m, Some n ⇒ Some (m + n) | _, _ ⇒ None end
  | Mul t1 t2 ⇒
    match card t1, card t2 with
    | Some 0, _ | _, Some 0 ⇒ Some 0
    | Some m, Some n ⇒ Some (m × n)
    | _, _ ⇒ None
    end
  | Fun t1 t2 ⇒
    match card t1, card t2 with
    | Some 0, _ ⇒ Some 1
    | _, Some 0 ⇒ Some 0
    | _, Some 1 ⇒ Some 1
    | Some m, Some n ⇒ Some (Nat.pow n m)
    | _, _ ⇒ None
    end
  end.

Lemma card_spec {t} :
  match card t with
  | Some n ⇒ ⟦t⟧ ≅ fin n
  | None ⇒
    inhabited ⟦t⟧ ∧
    (∀ k, fin k + ⟦t⟧ ≅ ⟦t⟧) ∧
    (∀ k, fin (S k) × ⟦t⟧ ≅ ⟦t⟧) ∧
    nat + ⟦t⟧ ≅ ⟦t⟧ ∧
    nat × ⟦t⟧ ≅ ⟦t⟧
  end.
